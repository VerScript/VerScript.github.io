VerScript Documentation

Welcome to the VerScript documentation. The language is built on a simple, spaced-function syntax that natively supports console IO, variable assignment, and mathematical evaluation.

1. The display Keyword

Output text, numbers, or variables directly to the terminal.

display "Hello World" display 100

2. Variables (var : val)

Variables are dynamically typed strings, integers, or booleans. Assign them using a colon.

name : "Alice" age : 25 is_active : true display name

3. Mathematical Operators & String Concatenation

VerScript supports standard arithmetic (+, -, *, /) seamlessly, as well as negative numbers and string concatenation.

score : 10 + -5 display "Score: " + score display 100 / 4

4. The prompt Keyword

Make your terminal scripts interactive by asking the user for input.

display "Enter your name: " prompt username display "Welcome, " display username

5. The loop Keyword

Repeat a block of code n times. The block of code must be indented using standard spacing (e.g. 2 spaces or a tab).

loop 3 display "This repeats 3 times"

6. The iterate Keyword

Iterate an index variable between specified numeric limits. You can customize the iteration index or omit the start limit (which defaults to the variable's current value, or 0 if undefined).

! Full syntax (inclusive limits): iterate i from 1 to 5 display i ! Omitted start limit (starts at current value of variable, or 0 if undefined): count : 3 iterate count to 6 display count

7. Comparison Operators

VerScript supports standard comparison operators that evaluate to booleans (true / false):

  • = (Checks if values are equal)
  • x= (Checks if values are not equal)
  • > (Greater than)
  • < (Less than)
  • >= (Greater than or equal to)
  • <= (Less than or equal to)
is_match : 10 = 10 display is_match ! Prints true is_diff : 5 x= 10 display is_diff ! Prints true

8. Conditionals (if-else)

Control flow using standard conditional statements. Like loops, conditional branches execute indented blocks of code.

val : 15 if val = 10 then display "Value is exactly 10" else if val > 10 then display "Value is greater than 10" else display "Value is less than 10"

9. Looping Conditionals (while & until)

Execute statement blocks dynamically based on expression checks:

  • while: Repeatedly runs while the condition evaluates to true.
  • until: Repeatedly runs until the condition evaluates to true (i.e. while it is false).
! While loop example count : 1 while count <= 3 display "count: " + count count : count + 1 ! Until loop example (runs until count2 is greater than 3) count2 : 1 until count2 > 3 display "until count2: " + count2 count2 : count2 + 1

10. Exception Handling & Conditional Fallbacks (do-unless)

The do-unless block supports both exception handling and conditional/watch execution.

Exception Handling

Catch runtime errors thrown by the compiler or interpreter. Errors can be caught by the generic keyword error or specific error types (such as DivisionByZeroError, UndefinedVariableError, InvalidOperandError, IndentationError, LoopIterationError, LoopLimitError, LoopDirectionError, or SyntaxError). Inside the unless block, the error keyword holds the name of the caught error.

do display "Trying division by zero..." val : 10 / 0 unless DivisionByZeroError display "Caught error: " + error do display "Accessing undefined variable..." display missing_val unless error display "Caught a general error: " + error

Conditional & Reactive Watch Execution

The condition block supports two execution modes: internal and external.

  • unless external [bool] (Default for booleans): Checks the condition beforehand. Once the do block starts, it runs to completion even if the condition is modified. If the condition was true initially, only the unless block is executed.
  • unless internal [bool] (Default for errors): Acts as a reactive watch. The condition is monitored after every statement inside the do block. If the condition becomes true at any point, execution of the do block stops immediately, and the unless block runs.

Example: Reactive Internal Watch

flag : false do display "Statement 1 runs" flag : true display "Statement 2 does NOT run" unless internal flag display "Watch triggered and unless block ran!"

Example: External Pre-check

flag : false do display "Statement 1 runs" flag : true display "Statement 2 runs (external watch does not stop it)" unless external flag display "This won't run because flag was false initially"

11. System Operators & Error Criticality (Error Points)

VerScript categorizes runtime errors into dynamic "Error Points" out of 10 to define their severity. Certain system block operators control how these errors are handled or skipped:

Error Criticality Levels (Error Points)

  • MemoryAllocationError (10 Points) - Critical: Thrown when dynamic memory allocation fails.
  • SystemError (9 Points) - Critical: Thrown on engine level limits, e.g. jump stack overflow.
  • SyntaxError (8 Points) - Critical: Thrown on structurally invalid statement layout.
  • IndentationError (8 Points) - Critical: Thrown on irregular or invalid statement indentation.
  • DivisionByZeroError (5 Points) - Moderate/Non-Critical: Thrown on dividing numbers by zero.
  • InvalidOperandError (4 Points) - Moderate/Non-Critical: Thrown on mismatching expression operand types.
  • LoopIterationError (3 Points) - Low/Non-Critical: Thrown on non-numeric loop bound arguments.
  • LoopLimitError (3 Points) - Low/Non-Critical: Thrown on non-numeric loop boundary ranges.
  • LoopDirectionError (3 Points) - Low/Non-Critical: Thrown when loop start bounds are greater than end limits.
  • UndefinedVariableError (2 Points) - Low/Non-Critical: Thrown on attempting to retrieve an uninitialized variable.

System Block Operators

  • ForceErrors: Disables error suppression and catch blocks inside its scope. Any error occurring within this block displays immediately and exits execution.
  • CriticalErrors: Only triggers halting/displaying for critical errors (8+ Error Points). Non-critical errors (1-7 Error Points) are ignored, and the faulty line is skipped.
  • SuppressErrors: Skips any statement line or block that triggers an error. Execution proceeds cleanly to the next sequential instruction.

Example: System Operators Showcase

! Suppressing all errors SuppressErrors val : 10 / 0 ! Error is silently skipped display "This line runs successfully!" ! Restricting to Critical errors CriticalErrors val2 : missing_val ! Non-critical UndefinedVariableError (2 Points) is skipped display "CriticalErrors bypassed moderate error!"

12. The throw Statement

VerScript allows manual triggering of exceptions. The target must be a valid registered error name. If an unregistered name is used, a SyntaxError is raised stating that the error name is invalid. You can also rethrow the active error inside an unless block via throw error.

do display "Throwing custom exception..." throw DivisionByZeroError unless DivisionByZeroError display "Caught thrown exception: " + error do display "Throwing invalid error..." throw InvalidErrorName unless SyntaxError display "Caught invalid throw: " + error
